Man In The Middle Attack

This attack, as it is called, is an attack in which the attacker manages to place himself between the service seeker (customer) and the service provider (server, router or other component) and thus can access the information and even change it without the user's knowledge. The most common way to attack this attack is through ARP SPOOFING. Our software scans the number of ARP RESPONSE facts and verifies the sender's IP address matches the MAC address, we will perform the test using the BROADCAST query of the sender's IP and thus we will perform the verification

SYN FLOOD Attack

This attack is actually a DDOS attack that prevents access to sites We can protect against this attack by setting rules in IPTABLES so that it is not observed in SYN requests.

DNS spoofing attack

This attack actually "poisons" the DNS cache and actually pulls a certain IP into the wrong domain. Our system knows how to detect this attack with the help of an algorithm that tests That we did not get more than one answer to a DNS query.

Port scanning attack

This is a method that attackers use to expand their target environment by sending packets to specific host ports and using responses to find vulnerabilities and understand which services and service versions are running on the attacked computer.
